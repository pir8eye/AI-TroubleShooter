## Pattern 1: Hair-Trigger on Security Software + GitHub Context

**Finding:** System constraint designed to prevent release of weaponized code triggered automatic deletion when user requested documentation for a private security repository.

**Evidence:**
- Report: [2026-01-06: Deletion Loop & Security Constraint Miscalibration](reports/2026-01-06-deletion-loop-security-constraint/)
- Trigger activated on: Security tool + GitHub repository context
- Response: Automatic deletion of generated content
- User's actual work: Legal, defensive security operations on own infrastructure

**Implication:** Constraint calibrated too broadly. Cannot distinguish between:
- Attack code (intended to block) vs. defensive tools (blocked incorrectly)
- Public releases (requires evaluation) vs. private internal repos (blocked incorrectly)
- Weaponized code vs. security operations documentation

## Cross-Pattern Observations

### Constraint Architecture Signature
- Automatic trigger on keyword/topic combination
- Executes at high speed before communication layers
- Operates independently of context layer
- No feedback loop or learning mechanism
- Disproportionate response (deletion vs. pause-and-communicate)

### User Impact
- Work destroyed without explanation
- Flow state broken
- Trust in system eroded
- Uncertainty about future reliability
- Secondary effects (self-doubt, chilling effect on future work)

### System Design Problem
- Overly broad trigger condition
- Wrong response mechanism for miscalibrated constraints
- No communication pathway
- Context-processing ignored
- No recalibration on repeated false positives
